# Dementia Exploratory Data Analysis

This project performs **Exploratory Data Analysis (EDA)** on global dementia healthcare indicators to understand patterns in diagnostics availability, healthcare workforce distribution, policy implementation, and care infrastructure across countries.

The analysis is implemented in Python using data analysis and visualization libraries such as **Pandas, NumPy, Matplotlib, and Seaborn**.

---

## Objectives

* Analyze global dementia healthcare indicators
* Study availability of diagnostic services
* Explore distribution of healthcare professionals involved in dementia care
* Identify policy and guideline implementation across countries
* Understand regional differences in dementia healthcare infrastructure

---

## Technologies Used

* Python
* Pandas
* NumPy
* Matplotlib
* Seaborn
* Jupyter Notebook

---

## Key Analyses

### Dementia Diagnostic Availability

Visualization of which countries report availability of dementia diagnostic services.

### Healthcare Workforce Distribution

Analysis of the number of neurologists, geriatricians, and psychiatrists involved in dementia care across countries.

### Policy and Guideline Implementation

Heatmap showing which countries have implemented dementia-related policies or clinical guidelines.

### Time Trend Analysis

Exploration of how dementia-related indicators have evolved over time.

### Regional Comparison

Comparison of dementia-related metrics across different global regions.

### Facility Availability

Visualization of dementia care facility availability by country.

---

## Visualizations Included

The notebook generates multiple visualizations such as:

* Country-level diagnostic availability plots
* Healthcare workforce distribution boxplots
* Policy implementation heatmaps
* Indicator trend plots
* Correlation matrices for healthcare indicators
* Regional distribution comparisons

## Dataset

The analysis uses publicly available datasets related to **global dementia healthcare indicators**.

Due to size constraints, the datasets are not included in the repository.
To run the notebook, place the dataset files inside a `data/` directory.
